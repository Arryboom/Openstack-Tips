    3.Openstack 数据库中文乱码问题

    openstack的sql connection 需要配置成utf-8才不会中文乱码
    1. sql_connection = mysql://nova:xxx@qa-mysql1:3306/nova?charset='utf-8'
    复制代码
    但是如果之前就没有设置utf-8，数据库又不能随便清空，想要更新openstack数据库某个字段，就不能直接采用utf-8的数据库连接，否则会出现插入到数据库中的是中文，但是openstack中显示的是乱码。更坏的是如果插入的是全角中文字符，openstack就会出错。

    那么如何更新这种情况下的数据库呢？

    答案很简单，就是采用和openstack一样的数据库连接方式，openstack使用的是 sqlalchemy 我写一个一个简单的数据库连接如下
    1. #!/usr/bin/python
    2. # coding=utf-8

    4. from sqlalchemy import create_engine
    5. from sqlalchemy import MetaData
    6. from sqlalchemy.sql import select
    7. from sqlalchemy.sql import update

    9. engine = create_engine('mysql://nova:password@qa-mysql1:3306/nova',convert_unicode=True)

    11. metadata=MetaData()
    12. meta=metadata
    13. meta.reflect(bind=engine)
    14. conn=engine.connect()
    15. secgroup=meta.tables['security_groups']

    17. u=update(secgroup).where(secgroup.c.id==60).values(name='中文')
    18. result=conn.execute(u)
    复制代码
    这里需要注意的是在create_engine的时候，需要添加convert_unicode=True,如果不添加，会出如下解码错误的错误
