openStack cinder存储域服务相关

  • |
  • 浏览：1892
  • |
  • 更新：2014-07-11 15:12
百度经验:jingyan.baidu.com

（一）在controller管理机上面配置cinder服务

cinder是块存储，用来给虚拟机挂扩展硬盘，就是将cinder创建出来的卷，挂到虚拟机里。

1、安装存储服务的包

# apt-get install cinder-api cinder-scheduler

2、编辑 /etc/cinder/cinder.conf配置文件，配置存储服务用的数据库路径

database]

...

connection = mysql://cinder:CINDER_DBPASS@controller/cinder

3、在mysql中创建cinder用户密码等相关信息

# mysql -u root -p

mysql> CREATE DATABASE cinder;

mysql> GRANT ALL PRIVILEGES ON cinder.* TO 'cinder'@'localhost'  IDENTIFIED BY 'CINDER_DBPASS'; 

mysql> GRANT ALL PRIVILEGES ON cinder.* TO 'cinder'@'%'  IDENTIFIED BY 'CINDER_DBPASS';

4、数据库同步

cinder-manage db sync

5、使用keystone为cinder创建验证服务

# keystone user-create --name=cinder --pass=CINDER_PASS--email=cinder@example.com

# keystone user-role-add --user=cinder --tenant=service --role=admin

6、在/etc/cinder/api-paste.ini编辑[filter:authtoken]选项

[filter:authtoken]

paste.filter_factory=keystoneclient.middleware.auth_token:filter_factory

auth_host=controller

auth_port = 35357

auth_protocol = http

auth_uri = http://controller:5000

admin_tenant_name=service

admin_user=cinder

admin_password=CINDER_PASS

7、编辑 /etc/cinder/cinder.conf配置文档，配置RabbitMQ消息队列服务

[DEFAULT]

...

rpc_backend = cinder.openstack.common.rpc.impl_kombu

rabbit_host = controller

rabbit_port = 5672

rabbit_userid = guest

rabbit_password = RABBIT_PASS

8、使用keystone创建存储服务并创建入口

# keystone service-create --name=cinder --type=volume --description="Cinder Volume Service"

# keystone endpoint-create \

--service-id=the_service_id_above\

--publicurl=http://controller:8776/v1/%\(tenant_id\)s \

--internalurl=http://controller:8776/v1/%\(tenant_id\)s \

--adminurl=http://controller:8776/v1/%\(tenant_id\)s

9、也可以创建第二个版本的keystone

# keystone service-create --name=cinderv2 --type=volumev2 --description="Cinder Volume Service V2"

# keystone endpoint-create \

--service-id=the_service_id_above\

--publicurl=http://controller:8776/v2/%\(tenant_id\)s \

--internalurl=http://controller:8776/v2/%\(tenant_id\)s \

--adminurl=http://controller:8776/v2/%\(tenant_id\)s

10、重启相关服务

# service cinder-scheduler restart

# service cinder-api restart

（二）存储节点机的系统环境设置

新添加一个用于存储的主机

1、配置/etc/hosts和/etc/hostname

2、安装mysql库

# apt-get install python-mysqldb

3、安装ntp并配置/etc/ntp，注释掉其他的server，并添加：

server 10.19.194.109

service ntp restart

4、安装Ubuntu云存档Havana，用于支持openstack稳定的ubuntu，然后更新ubuntu

（1）安装Havana

# apt-get install python-software-properties

# add-apt-repository cloud-archive:havana

（2）更新系统及重启

# apt-get update && apt-get dist-upgrade

# reboot

4、编辑 /etc/lvm/lvm.conf配置文档，修改device选项中的filter

5、配置完成后，安装存储服务包

# apt-get install cinder-volume

9、重启cinder服务

# service cinder-volume restart

# service tgt restart

（四）使用存储服务

1、在控制机上面通过cinder服务创建硬盘

root@controller:~# cinder create --display-name xx 10

2、在节点机上面查看该存储是否创建成功以及相关数据

3、使用 nova volume-attach命令将该硬盘挂载给虚拟机

#nova volume-attach cirrOS a6894ea7-096b-40d1-978e-1323482c4129 /dev/sdb1

然后会有相关提示，在web界面可以看到该设备已经挂到虚拟机cirros上面了

也可以进入虚拟机查看是否有存储挂到了虚拟机，发现硬盘/dev/sdb已经在虚拟机里面了，然后可在虚拟机使用该磁盘进行格式化分区等操作
